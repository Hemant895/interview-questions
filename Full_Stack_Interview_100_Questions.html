Part 1: Express.js (Q1 - Q25)

1. What is Express.js?
Express.js is a minimal and flexible Node.js web application framework that provides a robust set of features for web and mobile applications. It is the standard server framework for Node.js.
2. How is Express different from Node.js core http module?
The built-in http module requires writing a lot of boilerplate code to handle routing, requests, and responses. Express abstracts this away, providing a cleaner API for routing, middleware support, and JSON parsing.
3. What is Middleware in Express?
Middleware functions are functions that have access to the request object (`req`), the response object (`res`), and the `next` middleware function in the application’s request-response cycle. They can execute code, modify requests/responses, and end the cycle.
4. What is the role of `next()` in Express?
`next()` is a function that, when invoked, executes the next middleware in the stack. If not called, the request is left hanging unless a response is sent.
5. How do you handle 404 errors in Express?
app.use((req, res, next) => {
  res.status(404).send('Sorry, page not found!');
});
👉 Place this at the very end of all your routes.
6. Difference between `app.use()` and `app.get()`?
`app.use()` binds middleware to your application and runs for ALL HTTP methods (GET, POST, etc.) matching the path. `app.get()` only runs for HTTP GET requests on the exact specified route.
7. What is `express.json()` used for?
It is a built-in middleware function that parses incoming requests with JSON payloads. Without it, `req.body` will be undefined.
8. What is Express Router?
`express.Router` is a class to create modular, mountable route handlers. It acts as a "mini-application", keeping routes organized across multiple files.
9. How to enable CORS in Express?
By using the `cors` npm package: app.use(cors());. This allows your API to be requested from different domains.
10. Explain Error Handling Middleware in Express.
It has 4 arguments instead of 3: `(err, req, res, next)`. Express recognizes it as error-handling middleware. 
app.use((err, req, res, next) => {
  res.status(500).send('Something broke!');
});
11. What is the difference between `req.params` and `req.query`?
  • req.params: Route parameters (e.g., /users/:id). Extract id.
  • req.query: URL query strings (e.g., /users?sort=asc). Extract sort.
12. How do you securely store passwords in Express apps?
Using a hashing library like `bcrypt`. Never store plain text passwords. Always hash them before saving to the database.
13. What is JWT and how is it used in Express?
JSON Web Token is used for stateless authentication. In Express, you generate a JWT on login, send it to the client, and the client sends it back in the `Authorization` header on subsequent requests.
14. How to serve static files in Express?
Using the built-in middleware: app.use(express.static('public'));. This serves HTML, CSS, images, etc.
15. What is a templating engine in Express?
Engines like Pug, EJS, or Handlebars allow you to inject dynamic data into HTML files before sending them to the client.
16. What is `helmet` in Express?
Helmet is a middleware package that helps secure your Express apps by setting various HTTP headers to prevent attacks like XSS.
17. How do you handle file uploads in Express?
By using a middleware like `multer`. It parses `multipart/form-data` and saves the uploaded files to disk or memory.
18. What is the difference between `res.send()` and `res.json()`?
`res.send()` can send buffers, strings, objects, or arrays. `res.json()` explicitly formats the response as JSON and sets the appropriate headers.
19. How do you redirect in Express?
res.redirect('/new-path');
20. What is a reverse proxy and why use it with Express?
A server (like Nginx) that sits in front of your Node app. It handles SSL, caching, and load balancing, allowing Node to focus purely on application logic.
21. What is rate limiting?
Restricting the number of requests a user can make in a given timeframe to prevent abuse/DDoS. Implemented using packages like `express-rate-limit`.
22. How do you write a custom validation middleware?
Create a function that checks `req.body`, and if invalid, returns `res.status(400).json({error})`. Otherwise, calls `next()`.
23. How do you manage environments (dev/prod) in Express?
By using the `NODE_ENV` environment variable and packages like `dotenv` to load environment-specific configurations.
24. What are some performance optimizations for Express?
Use caching (Redis), gzip compression (`compression` package), avoid synchronous functions, and use clustering or PM2 to run multiple instances.
25. What is the difference between monolithic and microservice architecture in Node/Express?
Monolith has all application logic in one codebase/server. Microservices break the application into multiple independent Express apps communicating over HTTP or message queues.

Part 2: MongoDB (Q26 - Q50)

26. What is MongoDB?
MongoDB is a NoSQL, document-oriented database that stores data in flexible, JSON-like documents (BSON).
27. What is the difference between SQL and NoSQL databases?
SQL is relational, structured via tables/rows, and uses rigid schemas. NoSQL is non-relational, uses dynamic schemas, and stores data in documents, key-value pairs, or graphs.
28. What is a Document and a Collection in MongoDB?
A **Document** is a single record (like a row in SQL). A **Collection** is a group of documents (like a table in SQL).
29. What is Mongoose?
Mongoose is an Object Data Modeling (ODM) library for MongoDB and Node.js. It provides schema validation, models, and easy-to-use querying.
30. What is BSON?
Binary JSON. It is the binary encoding format MongoDB uses to store documents. It supports more data types than standard JSON (e.g., Dates, ObjectIds).
31. What is an ObjectId (`_id`)?
A 12-byte unique identifier automatically generated by MongoDB for every document to act as a primary key.
32. How do you enforce a Schema in MongoDB?
MongoDB is schema-less by default, but you can enforce validation rules at the database level, or use Mongoose in Node.js to define strict Schemas in application code.
33. What is an Index in MongoDB?
Indexes are special data structures that store a small portion of the collection's data in an easy-to-traverse form. They drastically improve query read performance but slow down writes.
34. What is the Aggregation Pipeline?
A framework for data processing. Documents enter a multi-stage pipeline (e.g., `$match`, `$group`, `$sort`) that transforms them into aggregated results.
35. `find()` vs `findOne()`?
`find()` returns a cursor to multiple documents (an array in Mongoose). `findOne()` returns exactly one document that matches the query.
36. What is the `$lookup` stage in Aggregation?
It performs a left outer join to an unsharded collection in the same database to filter in documents from the "joined" collection for processing. (MongoDB's version of a JOIN).
37. How to do transactions in MongoDB?
Since version 4.0, MongoDB supports multi-document ACID transactions across replica sets. You start a session, start a transaction, perform operations, and then commit or abort.
38. What is Replication?
The process of synchronizing data across multiple servers (Replica Sets) to provide redundancy, high availability, and failover.
39. What is Sharding?
A method for distributing data across multiple machines. It is MongoDB's approach to horizontal scaling for very large datasets.
40. What is the role of a primary and secondary node?
In a Replica Set, the Primary receives all write operations. Secondaries replicate the primary's oplog and apply operations to their data sets.
41. What is an `upsert`?
An operation that creates a new document if no documents match the filter criteria, or updates the document if a match is found.
42. How to back up a MongoDB database?
Using tools like `mongodump` for backups and `mongorestore` for restoration.
43. What is a covered query?
A query where all the requested fields are part of an index. MongoDB can return the result using only the index, without looking at the actual documents, making it extremely fast.
44. Explain `$set` vs `$push`.
`$set` updates the value of a specific field. `$push` appends a specified value to an array.
45. What are capped collections?
Fixed-size collections that support high-throughput operations that insert and retrieve documents based on insertion order. Once full, older documents are overwritten.
46. How do you delete a collection?
`db.collection.drop()`
47. What is GridFS?
A specification for storing and retrieving files that exceed the BSON-document size limit of 16 MB.
48. What is population in Mongoose?
The process of automatically replacing specified paths in a document with documents from other collections (similar to JOINs but done at the application layer).
49. Explain the `$match` aggregation stage.
Filters the document stream to allow only matching documents to pass unmodified into the next pipeline stage.
50. Can MongoDB replace Redis for caching?
While MongoDB is fast, Redis is an in-memory database and is generally much faster for volatile caching. MongoDB is a persistent data store.

Part 3: Object-Oriented Programming (Q51 - Q75)

51. What is Object-Oriented Programming (OOP)?
OOP is a programming paradigm based on the concept of "objects", which contain data (attributes) and code (methods). It aims to bind data and behaviors together.
52. What are the four main pillars of OOP?
1. Encapsulation
2. Abstraction
3. Inheritance
4. Polymorphism
53. What is a Class vs an Object?
A **Class** is a blueprint or template. An **Object** is an instance of a class containing actual values.
54. Explain Encapsulation.
The bundling of data and methods that operate on that data within a single unit (class), and restricting direct access to some of the object's components (using access modifiers like private/protected).
55. Explain Abstraction.
Hiding complex implementation details and showing only the essential features of the object to the user. (e.g., You know how to use a steering wheel without knowing how the engine works).
56. Explain Inheritance.
A mechanism where a new class (child) derives properties and behaviors from an existing class (parent). It promotes code reusability.
57. Explain Polymorphism.
The ability of different objects to respond to the same method call in their own specific way. "One interface, many methods."
58. What is Method Overloading?
Having multiple methods in the same class with the same name but different parameters (compile-time polymorphism). Note: JavaScript doesn't natively support this.
59. What is Method Overriding?
When a child class provides a specific implementation for a method that is already defined in its parent class (run-time polymorphism).
60. What is a Constructor?
A special method used to initialize objects when they are created.
61. What is the `this` keyword?
It refers to the current instance of the class. In JS, its value depends on how the function is called.
62. What is an Abstract Class?
A class that cannot be instantiated on its own and requires subclasses to provide implementations for its abstract methods.
63. What is an Interface?
A completely "abstract class" that contains only abstract methods. It is used to define a contract that implementing classes must follow.
64. Abstract Class vs Interface?
An abstract class can have both implemented and abstract methods and allows state (variables). An interface has only method signatures (no state or implementation) and a class can implement multiple interfaces.
65. What is Multiple Inheritance?
When a class inherits from more than one parent class. It can cause the "Diamond Problem" and is not supported in languages like Java or C# (they use interfaces instead).
66. What are Access Modifiers?
Keywords that set the accessibility of classes, methods, and properties: Public, Private, Protected.
67. What is a Static Method?
A method that belongs to the class itself rather than instances of the class. It can be called without creating an object.
68. What is Composition over Inheritance?
A principle stating that classes should achieve polymorphic behavior and code reuse by containing instances of other classes (composition) rather than inheriting from a base class.
69. What is a Singleton Pattern?
A design pattern that ensures a class has only one instance and provides a global point of access to it.
70. What is a Factory Pattern?
A design pattern that provides an interface for creating objects in a superclass, but allows subclasses to alter the type of objects that will be created.
71. What is the purpose of getters and setters?
They provide read/write access to private variables, allowing you to add validation logic during data manipulation (Encapsulation).
72. Explain SOLID principles.
S - Single Responsibility
O - Open/Closed
L - Liskov Substitution
I - Interface Segregation
D - Dependency Inversion
73. What is super()?
A keyword used in a child class constructor to call the constructor of its parent class.
74. What is a Destructor?
A method called automatically when an object is destroyed to free up resources. (Languages with garbage collection like JS/Java handle this automatically).
75. Is JavaScript an OOP language?
JavaScript is a multi-paradigm language that supports OOP. It uses Prototype-based inheritance under the hood, although modern JS provides `class` syntax as syntactic sugar.

Part 4: Data Structures & Algorithms (Q76 - Q100)

76. What is Big O Notation?
It describes the performance or complexity of an algorithm. It specifies the worst-case scenario regarding time (speed) or space (memory) as the input grows.
77. What is an Array?
A linear data structure consisting of a collection of elements stored at contiguous memory locations, allowing fast `O(1)` index access.
78. What is a Linked List?
A linear structure where elements (nodes) are not stored contiguously. Each node contains data and a pointer to the next node.
79. Array vs Linked List?
Arrays have fast reads `O(1)` but slow insertions/deletions `O(n)`. Linked Lists have slow reads `O(n)` but fast insertions/deletions `O(1)` if you have the node reference.
80. What is a Stack?
A linear structure that follows LIFO (Last In First Out). Think of a stack of plates. Operations: push, pop.
81. What is a Queue?
A linear structure that follows FIFO (First In First Out). Think of a line at a store. Operations: enqueue, dequeue.
82. What is a Hash Table (Hash Map)?
A data structure that maps keys to values for highly efficient lookup. It uses a hash function to compute an index. Average lookup time is `O(1)`.
83. What is a Collision in a Hash Table?
When the hash function maps two different keys to the same index. Handled via chaining (linked lists at the index) or open addressing.
84. What is a Tree?
A hierarchical non-linear data structure consisting of nodes connected by edges, starting from a root node.
85. What is a Binary Search Tree (BST)?
A tree where each node has at most two children. The left child's value is always less than the parent, and the right child's value is greater. Search time is `O(log n)`.
86. Tree Traversals (In-order, Pre-order, Post-order)?
- In-order: Left, Root, Right (Gives sorted order in BST)
- Pre-order: Root, Left, Right
- Post-order: Left, Right, Root
87. What is a Graph?
A non-linear structure consisting of vertices (nodes) and edges (lines connecting them). Can be directed or undirected.
88. BFS vs DFS in Graphs?
- Breadth-First Search (BFS): Explores neighbor nodes level by level. Uses a Queue.
- Depth-First Search (DFS): Explores as far as possible along a branch before backtracking. Uses a Stack.
89. What is Bubble Sort?
A simple sorting algorithm that repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order. Time: `O(n^2)`.
90. What is Merge Sort?
A divide-and-conquer algorithm that recursively divides the array into halves until they have one element, then merges them back in sorted order. Time: `O(n log n)`.
91. What is Quick Sort?
A divide-and-conquer algorithm that picks a 'pivot' element and partitions the array so elements smaller than the pivot come before it, and larger come after. Average Time: `O(n log n)`.
92. What is Binary Search?
A highly efficient search algorithm that finds an item in a **sorted** array by repeatedly dividing the search interval in half. Time: `O(log n)`.
93. What is Dynamic Programming (DP)?
An optimization technique that solves complex problems by breaking them down into simpler overlapping subproblems and storing their results (memoization) to avoid redundant computations.
94. Memoization vs Tabulation in DP?
Memoization is top-down (recursive with caching). Tabulation is bottom-up (iterative building of a table).
95. What is a Greedy Algorithm?
An algorithmic paradigm that builds up a solution piece by piece, always choosing the next piece that offers the most immediate benefit (local optimum).
96. What is a Heap?
A specialized tree-based data structure that satisfies the heap property. Max-Heap: Parent is greater than children. Min-Heap: Parent is smaller than children. Often used for Priority Queues.
97. What is a Trie?
A tree-like data structure used to efficiently store and retrieve keys in a dataset of strings. Great for autocomplete features.
98. How do you find a loop in a Linked List?
Using Floyd’s Cycle-Finding Algorithm (Tortoise and Hare). Use two pointers moving at different speeds. If they meet, there is a loop.
99. Time complexity of inserting in a Hash Table?
Average case: `O(1)`. Worst case: `O(n)` (if many collisions occur and they are chained in a single list).
100. What is Space Complexity?
The amount of memory an algorithm requires to run to completion, relative to the size of the input. Evaluated using Big O notation.

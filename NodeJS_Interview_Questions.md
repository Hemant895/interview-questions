# Node.js Interview Guide (Top 75 Questions)

## PART 1 (Q1–25)

### 1. What is Node.js? Where can you use it?
Node.js is a JavaScript runtime built on Chrome’s V8 engine that allows you to run JS outside the browser.

**Key Features:**
- Event-driven
- Non-blocking I/O
- Single-threaded with event loop

**Use Cases:**
- REST APIs
- Real-time apps (chat, live tracking)
- Streaming apps (video/audio)
- Microservices

<div class="tip">👉 Interview Tip: Say “I/O heavy apps, not CPU heavy”</div>

### 2. Node.js vs JavaScript

| Feature | JavaScript | Node.js |
|---|---|---|
| Type | Language | Runtime |
| Environment | Browser | Server |
| DOM Access | Has DOM | No DOM |
| Scope | Frontend | Backend |

<div class="tip">👉 Node = JS + APIs (fs, http, etc.)</div>

### 3. Why use Node.js?
- Handles thousands of concurrent requests
- Faster due to non-blocking architecture
- Same language (JS) everywhere
- Huge npm ecosystem

**Best for:** Chat apps, APIs, Streaming.

### 4. How does Node.js work?
Node works using:
- Call Stack
- Event Loop
- Callback Queue
- libuv (thread pool)

**Flow:** Request comes → Async task sent to libuv → Callback pushed to queue → Event loop executes

### 5. Is Node.js single-threaded?
Yes — main thread is single.

But it uses the **libuv thread pool** for:
- File system
- DNS
- Crypto

<div class="tip">👉 So effectively it is a hybrid model.</div>

### 6. How Node handles concurrency?
Using:
- Event loop
- Async operations
- Thread pool

<div class="tip">👉 No thread per request (unlike Java).</div>

### 7. What kind of API functions are supported?
- **Async** (preferred)
- **Sync** (blocking)

```javascript
fs.readFile()      // async
fs.readFileSync()  // blocking
```

### 8. What is a module in Node.js?
A reusable block of code.

**Types:**
- Core (`fs`, `http`)
- Local
- Third-party (`npm`)

```javascript
// math.js
exports.add = (a,b) => a+b;
```

### 9. What is npm?
Node Package Manager:
- Install libraries
- Manage dependencies
- Run scripts

```bash
npm install express
```

### 10. Why Node over Java/PHP?
- Non-blocking (better concurrency)
- Faster for I/O apps
- Lightweight

<div class="tip">❌ Not good for CPU-heavy tasks.</div>

### 11. What is middleware?
Functions executed between request & response.

**Uses:** Auth, Logging, Validation.

```javascript
app.use((req, res, next) => {
  console.log("Middleware");
  next();
});
```

### 12. What is control flow in Node.js?
Execution order of async tasks.
**Tools:** Callbacks, Promises, Async/await.

### 13. What is Event Loop?
Core mechanism that handles async operations and executes callbacks.
<div class="tip">👉 Runs continuously checking queue.</div>

### 14. Disadvantages of Node.js
- Not for CPU-heavy tasks
- Callback hell
- Single-thread limitation

### 15. What is REPL?
Interactive shell: Read Eval Print Loop.

```bash
node
> 5+5
10
```

### 16. Import & Export module
```javascript
// export
module.exports = fn;

// import
const fn = require('./file');
```

### 17. Node.js vs AJAX
| Feature | Node.js | AJAX |
|---|---|---|
| Layer | Backend | Frontend |
| Environment | Server-side | Client-side |

### 18. Popular DB with Node.js
- **MongoDB** (most common, fits JSON structure)
- MySQL
- PostgreSQL

### 19. Common libraries
Express, Mongoose, Lodash, Axios.

### 20. package.json
Project config file containing Dependencies, Scripts, Metadata.

### 21. Popular framework
<div class="tip">👉 Express.js (most used)</div>

### 22. What are Promises?
Handle async operations cleanly.

```javascript
fetchData()
  .then(res => console.log(res))
  .catch(err => console.log(err));
```

### 23. Event-driven programming
Execution based on events.
```javascript
emitter.on('event', () => {});
```

### 24. What is Buffer?
Used to handle binary data.
```javascript
const buf = Buffer.from("Hello");
```

### 25. What are Streams?
Process data in chunks.
**Types:** Readable, Writable, Duplex, Transform.

## PART 2 (Q26–50)

### 26. Explain crypto module in Node.js
Used for Encryption, Hashing, Secure data handling.

```javascript
const crypto = require('crypto');
const hash = crypto.createHash('sha256')
  .update('password')
  .digest('hex');
console.log(hash);
```
<div class="tip">👉 Used in: Password hashing, JWT signing.</div>

### 27. What is callback hell?
Nested callbacks → unreadable code.

```javascript
// Problem:
fs.readFile('a.txt', () => {
  fs.readFile('b.txt', () => {
    fs.readFile('c.txt', () => {});
  });
});
```
**Solution:** Promises, `async/await`.

### 28. Timers module in Node.js
Functions: `setTimeout`, `setInterval`, `setImmediate`.

### 29. setImmediate() vs process.nextTick()
| `setImmediate` | `nextTick` |
|---|---|
| Runs later | Runs immediately |
| After I/O | Before event loop |

<div class="tip">👉 `nextTick` has higher priority.</div>

### 30. setTimeout() vs setImmediate()
- `setTimeout(fn,0)` → next cycle
- `setImmediate(fn)` → after I/O
Order may vary depending on context.

### 31. What is fork in Node.js?
Creates child process (Node specific).
```javascript
const { fork } = require('child_process');
fork('child.js');
```

### 32. spawn() vs fork()
| `spawn` | `fork` |
|---|---|
| Runs any process | Only Node |
| No IPC by default | Has IPC |

### 33. Passport module
Authentication middleware supporting JWT, OAuth, Local login.

### 34. Avoid callback hell (3 ways)
1. Promises
2. Async/Await
3. Modular code

### 35. EventEmitter
Handles events.
```javascript
const EventEmitter = require('events');
const emitter = new EventEmitter();

emitter.on('msg', () => console.log("Hello"));
emitter.emit('msg');
```

### 36. URL module
Parse URLs.
```javascript
const url = new URL('https://abc.com?a=1');
console.log(url.searchParams.get('a'));
```

### 37. Install, update, delete dependency
```bash
npm install pkg
npm update pkg
npm uninstall pkg
```

### 38. React Portals (Node context)
Used in frontend (React), not Node directly.
<div class="tip">👉 Render outside DOM hierarchy.</div>

### 39. What is piping?
Stream chaining.
```javascript
readStream.pipe(writeStream);
```

### 40. File flags (read/write)
- `r` → read
- `w` → write
- `a` → append

### 41. Open file in Node.js
```javascript
fs.open('file.txt', 'r', callback);
```

### 42. Reactor pattern
Handles I/O events asynchronously.
<div class="tip">👉 Core idea behind Node event loop.</div>

### 43. Test pyramid
1. Unit tests (most)
2. Integration tests
3. E2E tests (least)

### 44. body-parser
Parses request body.
```javascript
app.use(express.json());
```

### 45. CORS
Cross-Origin Resource Sharing. Allows frontend to call backend from different domain.

### 46. TLS module
Secure communication (HTTPS).

### 47. What is cluster?
Runs Node on multiple CPU cores.
```javascript
const cluster = require('cluster');
```

### 48. Manage sessions
- Cookies
- express-session
- Redis

### 49. Types of streams
Readable, Writable, Duplex, Transform.

### 50. Authentication & Authorization
- **Authentication:** Who you are (login).
- **Authorization:** What you can access.
*(Example: JWT, OAuth)*

## PART 3 (Q51–75)

### 51. Packages for file uploading
Common packages: `multer` (most used), `formidable`, `busboy`.

```javascript
// Example (Multer):
const multer = require('multer');
const upload = multer({ dest: 'uploads/' });

app.post('/upload', upload.single('file'), (req, res) => {
  res.send("File uploaded");
});
```

### 52. Handle DB connections
Use Connection pooling, ORM/ODM (Mongoose, Sequelize).
```javascript
mongoose.connect(DB_URL, { useNewUrlParser: true });
```
<div class="tip">👉 Avoid opening connection per request.</div>

### 53. Read command line arguments
```javascript
console.log(process.argv);
// node app.js hello
// Output: ['node','app.js','hello']
```

### 54. Child processes
Run parallel tasks using: `spawn`, `fork`, `exec`.
<div class="tip">👉 Used for CPU-heavy work.</div>

### 55. NODE_ENV
Environment variable: `development`, `production`, `test`.
```javascript
if (process.env.NODE_ENV === 'production') { }
```

### 56. Timing features
- `setTimeout`
- `setInterval`
- `setImmediate`
- `process.nextTick`

### 57. What is WASI?
WebAssembly System Interface.
<div class="tip">👉 Allows running WebAssembly outside browser (Node support).</div>

### 58. First-class function
Functions treated as values: Pass as argument, Return from function.
```javascript
function greet(fn) {
  fn();
}
```

### 59. Node vs other frameworks
**Node advantages:** Fast I/O, Lightweight, Event-driven.

### 60. What is stub?
Fake function used in testing.
<div class="tip">👉 Replaces real implementation.</div>

### 61. What is libuv?
Core library behind Node.js. Handles Event loop, Thread pool, Async I/O.

### 62. Enhance performance using clustering
Use multiple CPU cores:
```javascript
cluster.fork();
```
<div class="tip">👉 Improves scalability.</div>

### 63. When NOT to use Node.js
Avoid when: CPU-heavy tasks, Complex computations, ML processing.

### 64. Debug Node.js
Tools: `node inspect`, Chrome DevTools, VS Code debugger.

### 65. What is N-API?
Allows building native addons (C/C++).
<div class="tip">👉 Stable API for Node extensions.</div>

### 66. Domain module
Used for error handling (deprecated now).
<div class="tip">👉 Replaced by better patterns (try/catch, async handling).</div>

### 67. Graceful shutdown
Close server properly:
```javascript
process.on('SIGINT', () => {
  server.close(() => {
    console.log("Closed");
  });
});
```

### 68. dependencies vs devDependencies vs peerDependencies
| Type | Purpose |
|---|---|
| `dependencies` | Needed in production |
| `devDependencies` | Dev only (testing, build) |
| `peerDependencies` | Required by library user |

### 69. Redis module
Used for: Caching, Sessions, Pub/Sub.

### 70. What is WebSocket?
Real-time communication (bi-directional).
<div class="tip">👉 Used in: Chat apps, Live updates.</div>

### 71. util module
Provides utility functions: `promisify`, `format`, `inspect`.

### 72. Environment variables
Access using `process.env.KEY`. Use `.env` file (`dotenv`).

### 73. DNS module
Used for domain resolution:
```javascript
dns.lookup('google.com', callback);
```

### 74. Tracing in Node.js
Used for Performance monitoring, Debugging.
<div class="tip">👉 Tools: OpenTelemetry, APM tools.</div>

### 75. Thread pool in Node.js
Handled by libuv. Default size: 4 threads.
Used for: File system, Crypto, DNS.
<div class="tip">👉 Can increase: `UV_THREADPOOL_SIZE=8`</div>

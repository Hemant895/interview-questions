Node.js Interview Guide (Top 75 Questions)

PART 1 (Q1–25)

1. What is Node.js? Where can you use it?
Node.js is a JavaScript runtime built on Chrome’s V8 engine that allows you to run JS outside the browser.

Key Features: Event-driven, Non-blocking I/O, Single-threaded with event loop.
Use Cases: REST APIs, Real-time apps (chat, live tracking), Streaming apps (video/audio), Microservices.
👉 Interview Tip: Say “I/O heavy apps, not CPU heavy”

3. Why use Node.js?
- Handles thousands of concurrent requests
- Faster due to non-blocking architecture
- Same language (JS) everywhere
- Huge npm ecosystem

Best for: Chat apps, APIs, Streaming.
4. How does Node.js work?
Node works using: Call Stack, Event Loop, Callback Queue, libuv (thread pool).

Flow: Request comes → Async task sent to libuv → Callback pushed to queue → Event loop executes.
5. Is Node.js single-threaded?
Yes — main thread is single.
But it uses the libuv thread pool for: File system, DNS, Crypto.
👉 So effectively it is a hybrid model.
6. How Node handles concurrency?
Using: Event loop, Async operations, Thread pool.
👉 No thread per request (unlike Java).
7. What kind of API functions are supported?
- Async (preferred)
- Sync (blocking) 
fs.readFile()      // async
fs.readFileSync()  // blocking
8. What is a module in Node.js?
A reusable block of code.
Types: Core (fs, http), Local, Third-party (npm). 
// math.js

9. What is npm?
Node Package Manager: Install libraries, Manage dependencies, Run scripts. 
npm install express
10. Why Node over Java/PHP?
- Non-blocking (better concurrency)
- Faster for I/O apps
- Lightweight
❌ Not good for CPU-heavy tasks.
11. What is middleware?
Functions executed between request & response. Uses: Auth, Logging, Validation.

  console.log("Middleware");
  next();
});

Execution order of async tasks. Tools: Callbacks, Promises, Async/await.
13. What is Event Loop?
Core mechanism that handles async operations and executes callbacks.
👉 Runs continuously checking queue.

👉 Used in: Password hashing, JWT signing.
27. What is callback hell?
Nested callbacks → unreadable code. 
// Problem:
fs.readFile('a.txt', () => {
  fs.readFile('b.txt', () => {
    fs.readFile('c.txt', () => {});
  });
});
Solution: Promises, async/await.
